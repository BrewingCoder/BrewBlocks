# BrewBlocks — orientation for AI agents

**Project type:** Decorative stone + armor gadget mod for Minecraft, ported from Forge 1.15.2 to NeoForge 1.21.1. Personal / small modpack use; MIT-equivalent license. Owner: Scott Singleton (GitHub: BrewingCoder).

## What's shipped (as of f8bab2c — 2026-04-20)

### Blocks (136 total)

- **1 ore:** `sticky_ore` — infested, spawns slime on break; chance to spawn creeper instead (config-gated).
- **3 stone families** × 5 variants × 9 shapes = 135:
  - Families: `abyssal` (near-black), `quarried` (near-white), `burnt` (warm brown)
  - Variants per family: `<fam>_stone`, `<fam>_bricks`, `<fam>_bricks_cracked`, `<fam>_cobblestone`, `<fam>_smooth_stone`
  - Shapes per variant: base cube, stairs, slab, wall, fence, fence_gate, button, pressure_plate, pillar (RotatedPillarBlock, axis-rotatable)
- **Naming convention:** vanilla-consistent. `abyssal_stone_stairs` / `abyssal_cobblestone_wall` / `abyssal_smooth_stone_pillar` — full base name suffix. **Brick** shapes use singular `_brick_` (mirroring vanilla `stone_brick_slab`): `abyssal_brick_stairs`, `abyssal_brick_cracked_wall`.

### Items

- **Night Vision Goggles** (`goggles`) — helmet. Grants NV while worn + **-10 MAX_HEALTH** debuff (matches 1.15 "significantly reduce health" README). Effects owned by `GogglesEffectsHandler` (global `PlayerTickEvent.Post`), covering both head armor slot and head curio slot in one place.
- **Slime Bounce Boots** (`slime_bounce_boots`) — feet. `LivingFallEvent` handler: if fall distance > 2 and not sneaking, cancel fall damage, invert vertical motion × 0.9, apply horizontal friction reduction (0.95), damage boots by 10, play slime-jump sound, register with `PlayerBounceHandler` for tick follow-through.
  - Sneaking + fall > 2 = damageless landing.
  - `PlayerBounceHandler` maintains an `IdentityHashMap<Entity, Handler>` keyed on the player. `PlayerTickEvent.Post` applies the bounce impulse, preserves horizontal momentum, cleans up 5 ticks after landing.
- Both items work in vanilla armor slot AND Curios head/feet slot (can wear alongside regular armor).

### Recipes (258 total)

Owned by `ModRecipeProvider`.
- **Stonecutter** — 1:1 (slab 1:2) from each base to all 8 shape variants; plus 1:1 base↔bricks↔smooth within family. 74 stonecutter recipes at the current count; rest is crafting/smelting.
- **Shaped crafting** — stairs (3×3 descending), slab (horizontal row), wall (bottom row), fence, fence_gate, button (1→1), pressure_plate (2→1), pillar (2 stacked → 2 pillars, vanilla quartz pattern).
- **Smelting chain** — cobblestone → stone → smooth_stone, bricks → bricks_cracked. 2×2 crafting for stone → bricks.

> Local helper `stonecut(...)` writes recipes to the mod namespace. **Do not use vanilla `stonecutterResultFromBase`** — it silently saves into `data/minecraft/recipe/` because its internal `save(out, String)` defaults to the `minecraft` namespace when given a bare string.

### World generation (data-driven)

All three stone families + sticky ore spawn as veins in overworld biomes:
- `data/brewblocks/worldgen/configured_feature/<name>.json` — ore feature targeting `minecraft:stone_ore_replaceables` + `minecraft:deepslate_ore_replaceables`
- `data/brewblocks/worldgen/placed_feature/<name>.json` — count 10/chunk, `in_square`, y=0–63 uniform, biome filter
- `data/brewblocks/neoforge/biome_modifier/add_<name>.json` — `#minecraft:is_overworld`, step `underground_ores`
- Cluster sizes: 5 for sticky ore, 25 for the three stone families (matches old 1.15 config defaults)

### Textures

- **Base mod** ships **16× textures** at `assets/brewblocks/textures/block/` — generated by `tools/regen_16x.py` from the 64× HD set using a per-channel retint of vanilla stone variants.
- **HD resource pack** (`brewblocks_hd`) bundled in-jar at `resourcepacks/brewblocks_hd/` with **64× textures** and its own `pack.mcmeta` (pack_format 34). Toggleable via the in-game Resource Packs screen (`AddPackFindersEvent` wires it as `PackSource.BUILT_IN`, `alwaysActive=false`, `Pack.Position.TOP`).
- **Armor layer textures** (64×32) live at `assets/brewblocks/textures/models/armor/{goggles,slime}_layer_1.png` — that's the vanilla armor-layer resolution, NOT an upscale. Do not move these into the HD pack.
- **Item textures** `goggles.png` and `slime_bounce_boots.png` are 16×16 at `textures/item/`.

### Curios integration (soft dep)

- **All** `top.theillusivec4.*` imports confined to `com.brewingcoder.brewblocks.curios/` subpackage.
- Main class guards loading with `if (ModList.get().isLoaded("curios"))` — JVM never touches the subpackage when Curios is absent, so the mod loads fine standalone.
- `CuriosIntegration.init(modBus)` listens for `RegisterCapabilitiesEvent` and registers `CuriosCapability.ITEM` for goggles + boots.
- Slot whitelist is data-driven: `data/curios/tags/items/head.json` (goggles) and `data/curios/tags/items/feet.json` (boots). Slot availability comes from `data/brewblocks/curios/entities/player.json` which adds `head` + `feet` slots to the player.
- `CuriosSlots.isWornAsCurio(entity, item)` + `findFirstCurio(entity, item)` are the only runtime touchpoints — used from `GogglesEffectsHandler` and `SlimeBounceBootsItem.Events.findBoots()`, each guarded by a `CURIOS_LOADED` static boolean.

### Config (`brewblocks-common.toml`)

`ModConfigSpec` with a `sticky_ore` section:
- `doesStickyOreSpawnSlime` (default true)
- `doesStickyOreChanceSpawnCreeper` (default true)
- `stickyOreSpawnCreeperChance` (default 5, range 0–100)

## Where to look in code

- `registry/` — four `DeferredRegister`s (Blocks, Items, Tabs, ArmorMaterials). `ModBlocks.java` is the biggest file (~265 lines); registration helpers `plain/stair/slab/wall/fence/fenceGate/button/plate/pillar` at the bottom.
- `block/StickyOreBlock` — extends `DropExperienceBlock` with xp 0–2; overrides `playerDestroy` to maybe-spawn a Slime or Creeper on server side.
- `item/` — `NightVisionGogglesItem` + `SlimeBounceBootsItem` (thin wrappers), `GogglesEffectsHandler` (owns the goggles' NV + max-health debuff, single source of truth for both wear paths), inner `Events` class on the boots (fall handler).
- `entity/PlayerBounceHandler` — bounce tick follow-through, static map registry, inner `Events` class subscribed to `PlayerTickEvent.Post`.
- `config/` — `Configs.java` + `WorldConfig.java` (`ModConfigSpec`).
- `data/` — 7 data providers: `DataEvents` (GatherDataEvent dispatcher), `ModBlockStateProvider`, `ModItemModelProvider`, `ModBlockTagsProvider`, `ModItemTagsProvider`, `ModBlockLootProvider` (+ `ModLootTableProvider` wrapper), `ModRecipeProvider`.
- `resources/HDResourcePack` — the `AddPackFindersEvent` handler for the built-in HD pack.
- `curios/` — isolated soft-dep: `CuriosIntegration`, `GogglesCurio`, `BootsCurio`, `CuriosSlots`.

## Common workflows

### Build & verify
```bash
./gradlew build --console=plain        # compiles + assembles jar under build/libs/
./gradlew runData --console=plain      # regenerate src/generated/resources (blockstates, models, tags, loot, recipes)
./gradlew runClient --console=plain    # launch dev client — NOT YET VERIFIED at runtime
```

### Regenerate 16× textures from HD
```bash
python3 tools/regen_16x.py
```
Pulls vanilla stone/cobblestone/stone_bricks/cracked_stone_bricks/smooth_stone/coal_ore from the cached Minecraft client jar (`~/.gradle/caches/neoformruntime/artifacts/minecraft_1.21.1_client.jar`) and rescales each channel by `target_mean / base_mean` so the output's mean color matches the HD texture's mean. Requires Pillow (`brew install pillow`).

### Adding a new block shape variant
1. Add the `DeferredBlock<T>` field + helper call in `ModBlocks.java`.
2. Add the shape to `decoSet(...)` args in `ModBlockStateProvider.java` + `ModRecipeProvider.java`.
3. Update the tag-dispatch `if/else` chain in `ModBlockTagsProvider` if the new shape has its own tag.
4. Add lang entries to `assets/brewblocks/lang/en_us.json` (yes, still hand-maintained — no `LanguageProvider` yet).
5. `./gradlew runData && ./gradlew build`.

### Adding a new stone family
Look at how `burnt` was added in commit `749777c`. The shape: copy one 64× texture per variant (5 total) into `resourcepacks/brewblocks_hd/assets/brewblocks/textures/block/<family>_<variant>.png`, add the family to `MAPPING` in `tools/regen_16x.py`, run the script, then mirror the abyssal/quarried registry + provider + recipe + worldgen + lang structure.

## Gotchas — things that bit during the port

- **`BlockTagsProvider` import:** use `net.neoforged.neoforge.common.data.BlockTagsProvider` (takes PackOutput + modid + ExistingFileHelper), **not** the vanilla class which doesn't accept EFH.
- **Stonecutter recipe namespace:** vanilla `stonecutterResultFromBase(...)` writes to `data/minecraft/recipe/`, not your mod namespace. Local `stonecut(...)` helper in `ModRecipeProvider` forces the right namespace via explicit `ResourceLocation.fromNamespaceAndPath(...)`.
- **ArmorItem ATTRIBUTE_MODIFIERS override:** the super constructor sets the component from material defaults, overwriting anything you put in Properties. Can't add custom attribute modifiers via Item.Properties alone. Workaround: apply/remove the modifier from a `PlayerTickEvent` handler (see `GogglesEffectsHandler`). Idempotent using a stable `ResourceLocation` as the modifier ID.
- **Curios classloader safety:** any class that `import`s a `top.theillusivec4.*` type MUST only be referenced behind `ModList.get().isLoaded("curios")`. The `curios/` subpackage is the only place these imports live; every entry point is guarded.
- **HD pack register path:** `event.addPackFinders(BrewBlocks.id("resourcepacks/brewblocks_hd"), ...)` — the `ResourceLocation` path is the folder relative to resources root, not a namespace-scoped lookup. Don't put `/` before the path.
- **Nested `pack.mcmeta` required:** the HD pack won't load without its own `pack.mcmeta` at `resourcepacks/brewblocks_hd/pack.mcmeta` using `pack_format: 34`. A missing/wrong-version nested mcmeta shows the red "wrong version" badge and the pack won't auto-enable even with `alwaysActive=true`.
- **`inventoryTick` fires for every slot** the item is in, not just the equipped one. Guard with `player.getItemBySlot(getEquipmentSlot()) == stack` — but since goggles effects moved to the global `PlayerTickEvent` handler, this isn't an issue anymore. Worth remembering for new items.
- **`@EventBusSubscriber` auto-scans.** Putting it on a class that imports `top.theillusivec4.*` types will crash at mod-load time when Curios is absent. The `curios/CuriosIntegration` class deliberately does NOT use `@EventBusSubscriber` — it's registered manually via `modBus.addListener(...)` from inside the ModList guard.

## Deferred / known gaps

- **No `runClient` smoke test yet.** `FenceGateBlock(WoodType.OAK, ...)` compiles but the sound effect is weird for stone gates — probably acceptable until someone complains. `BlockSetType.STONE` buttons + pressure plates compile fine; visual verification pending.
- **Armor recipes** — goggles + slime boots have no crafting recipe. Currently creative-only.
- **Pillar textures** — all pillars reuse the base stone texture on every face (via `axisBlock(block, tex, tex)`). No dedicated pillar-band texture exists; pillars visually look like rotatable cubes until bespoke art lands.
- **Flax crop** — deliberately dropped per owner decision ("plenty of other mods do flax now"). Old lang entries + textures purged.
- **CurseGradle publishing** — `com.matthewprenger.cursegradle` plugin was dropped in the port; wire back when ready to publish to CurseForge. Also no `maven-publish` block configured for the personal maven.
- **JEI / TheOneProbe / CraftTweaker / Mekanism integrations** — all dropped. Re-add selectively if a downstream modpack needs it.
- **.gitattributes** — the `forge-1.15.2` branch has one commit master doesn't. Cherry-pick if it matters.
